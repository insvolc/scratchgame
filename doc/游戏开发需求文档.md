# 刮刮乐模拟游戏开发需求文档

## 1. 项目概述

### 1.1 项目名称
《幸运刮刮乐》（暂定）

### 1.2 游戏类型
休闲模拟类游戏，核心玩法为模拟真实彩票刮刮乐的开奖体验。

### 1.3 游戏目标
让玩家在虚拟环境中体验购买、刮奖、中奖的完整流程，通过金币系统建立经济循环，提供轻松解压的娱乐体验。

### 1.4 目标用户
- 年龄：18 岁以上
- 偏好：休闲、轻竞技、模拟体验类玩家
- 场景：碎片化时间娱乐

---

## 2. 发布平台

| 平台 | 技术方案建议 | 备注 |
|---|---|---|
| Web 端 | H5 页面 / Web App | 支持浏览器访问，可部署为 PWA |
| 微信小程序 | 微信小程序原生 / Taro / uni-app | 需接入微信登录、支付（虚拟货币） |
| 抖音小程序 | 抖音小程序框架 / Taro | 需遵循抖音平台审核规范 |

> 建议采用跨端方案（如 Taro）统一开发，降低多端维护成本。

---

## 3. 核心玩法

### 3.1 主循环
1. 玩家进入游戏，获得初始金币
2. 在商城选择并购买刮刮乐彩票
3. 购买的彩票进入背包
4. 玩家从背包选择彩票进入刮奖页面
5. 模拟刮奖动作，揭晓中奖结果
6. 中奖金币自动发放到账户

### 3.2 刮奖体验
- 彩票表面覆盖刮奖涂层
- 玩家通过手指/鼠标滑动进行刮除
- 涂层刮除比例达到阈值后自动全部揭开
- 揭开时播放音效和动画反馈

---

## 4. 系统需求

### 4.1 金币系统

| 功能 | 说明 |
|---|---|
| 金币获取 | 初始赠送、每日签到、任务奖励、中奖奖励、观看广告奖励 |
| 金币消耗 | 购买彩票 |
| 金币显示 | 主界面顶部常驻显示当前金币数量 |
| 数据持久化 | 金币数量需本地存储并同步云端（登录后） |

### 4.2 彩票商城

- 展示可购买的刮刮乐彩票列表
- 每张彩票显示：
  - 名称
  - 售价（金币）
  - 最高奖金
  - 中奖概率概览
  - 主题封面
- 点击购买时判断金币是否足够
- 支持单次购买和批量购买

### 4.3 背包系统

- 显示玩家已购买但未刮开的彩票
- 支持按彩票类型筛选
- 点击彩票进入刮奖页面
- 已刮开的彩票可设置保留或自动清理

### 4.4 刮奖页面

- 彩票完整展示
- 涂层覆盖中奖区域
- 支持手势/鼠标刮除
- 实时计算刮除比例
- 中奖后高亮显示中奖区域
- 显示获得金币数额
- 提供"再刮一张"和"返回"按钮

### 4.5 中奖系统

- 每张彩票预先生成中奖结果
- 中奖等级可分为：
  - 特等奖
  - 一等奖
  - 二等奖
  - 三等奖
  - 四等奖
  - 未中奖
- 中奖结果由概率表控制
- 开奖过程公平、结果在购买或生成时已确定

## 5. 彩票类型设计

参考中国福利彩票和体育彩票常见刮刮乐类型，设计以下虚拟彩票：

### 5.1 经典数字匹配型
- 玩法：刮开"我的号码"和"中奖号码"，匹配数字获得对应奖金
- 参考：福彩"好运十倍"、体彩"甜蜜蜜"

### 5.2 对奖符型
- 玩法：刮开区域，出现指定图标即中奖
- 参考：体彩"点石成金"

### 5.3 九宫格/三同图型
- 玩法：刮开九宫格，三个相同图案连成一线即中奖
- 参考：福彩"红楼探秘"

### 5.4 幸运数字型
- 玩法：刮开一个数字，直接显示中奖金额
- 简单直接，适合新手

### 5.5 多区域混合型
- 玩法：一张彩票包含多个小游戏区域
- 提升丰富度和耐玩性

## 6. UI/UX 需求

### 6.1 主界面
- 顶部：金币数量、设置、任务入口
- 中部：商城入口、背包入口
- 底部：个人中心、排行榜（可选）

### 6.2 商城界面
- 网格/列表展示彩票
- 筛选标签：全部、低价、高价、热门
- 购买确认弹窗

### 6.3 背包界面
- 已购彩票缩略图
- 未读/未刮标记
- 空状态提示

### 6.4 刮奖界面
- 全屏沉浸式体验
- 刮涂层动画流畅
- 中奖弹窗庆祝效果
- 未中奖温和反馈

### 6.5 通用要求
- 界面风格：明快、喜庆、金币/红色/金色为主色调
- 动画：购买、刮奖、中奖均需动画反馈
- 音效：刮擦声、中奖欢呼声、金币掉落声
- 提供关闭音效选项

## 7. 技术需求

### 7.1 跨端适配
- 统一代码库，适配 Web、微信小程序、抖音小程序
- 响应式布局，适配手机和平板
- 触摸事件与鼠标事件兼容

### 7.2 刮奖涂层实现
- 使用 Canvas 绘制涂层
- 支持多点触控
- 刮除区域像素比例计算
- 性能优化，避免卡顿

### 7.3 数据存储
- 本地：金币、背包、设置等使用本地缓存
- 云端：登录后同步用户数据
- 离线可玩，联网后同步

### 7.4 安全性
- 中奖结果不应在客户端直接暴露
- 关键逻辑建议服务端校验（如需真随机）
- 防作弊：金币数量校验、背包校验

### 7.5 性能要求
- 首屏加载时间 < 3 秒
- 刮奖帧率 > 30fps
- 内存占用合理，避免小程序闪退

## 9. 平台合规与注意事项

### 9.1 防沉迷与适龄
- 游戏内明确标注"模拟游戏，不涉及真实金钱赌博"
- 虚拟货币不可兑换真实货币
- 建议设置 18 岁以上提示

### 9.2 小程序审核
- 避免使用真实彩票品牌名称和图案
- 不得涉及真实赌博
- 虚拟金币系统需符合各平台规范

### 9.3 广告与变现
- 可提供观看广告获取金币的选项
- 广告奖励需合理，不影响核心体验
